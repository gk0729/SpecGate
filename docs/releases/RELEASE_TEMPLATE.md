## Quick SemVer Rule

- PATCH (`v1.0.1`): bug fixes, no behavior break
- MINOR (`v1.1.0`): backward-compatible feature additions
- MAJOR (`v2.0.0`): breaking interface/behavior changes

## 60-Second Publish Checklist

1. 建立並推送 tag（例如 `v1.0.1`）
2. 開 GitHub Release 並選對 tag
3. 貼上本範本、補齊內容
4. 確認是否為 `Latest`（通常是）
5. 按下 Publish release
